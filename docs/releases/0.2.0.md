# BoatLock Firmware 0.2.0 Release Notes

Release date: 2026-03-02

## Highlights

- Added deterministic on-device HIL simulation with closed-loop execution and report API over BLE (`SIM_*` commands).
- Expanded built-in regression scenario set to `S0..S19`, including randomized current/noise and hardware-failure emulation.
- Hardened BLE transport for long-running HIL sessions (connection params, queued command/log handling, notify gating).
- Fixed reboot under large `SIM_REPORT` payloads and fixed repeated GNSS jump event storms.
- Added sticky simulation badges on device UI (`SIM LIVE`, post-run `SIM <id> PASS/FAIL`).

## New Failure Emulation Coverage

- Compass signal loss/restore (`COMPASS_LOST_EMU`, `COMPASS_RESTORED_EMU`)
- Power loss/restore (`POWER_LOSS_EMU`, `POWER_RESTORED_EMU`)
- Display loss/restore (`DISPLAY_LOST_EMU`, `DISPLAY_RESTORED_EMU`)
- Actuator derating windows (`ACTUATOR_DERATE_EMU`, `ACTUATOR_DERATE_CLEAR_EMU`)

## Validation Snapshot

- Native HIL test suite: pass
  - `pio test -e native -f test_hil_sim -vv`
- Firmware build (`esp32s3`): pass

- Hardware quick regression: pass
